# 版本记录：v1.1-profile-seeding

## 1. 本轮目标

本轮目标是在已经完成技术基线的前提下，把站点从“示例内容展示”推进到“真实人物资料接入”的第一阶段。

具体目标：

- 用用户已提供的真实身份信息替换首页示例文案
- 把项目、投资、思考模块改成诚实的空态，而不是继续使用虚构占位案例
- 保持双语结构不变，继续为后续补全真实内容预留扩展空间

这一轮属于：内容替换 + 结构修正 + 可维护性提升。

## 3. 问题分析

上一轮的核心问题已经解决，但引出了新的内容真实性问题：

1. 站点结构虽然成熟，但其中的项目、投资、思考示例会造成“像是已经拥有这些内容”的误导。
2. 如果继续保留示例文章与案例，页面会更丰富，但会削弱用户身份的真实性。
3. 个人网站早期最重要的不是“看起来内容很多”，而是“让已知信息准确且有质感地落地”。

这意味着本轮的核心矛盾不再是技术选型，而是：

- 页面完整度
- 内容真实性
- 后续可持续扩展性

三者之间的平衡。

## 4. 候选方案比较

### 方案 A：保留示例内容，等以后再替换

核心思路：
维持当前页面充实度，等用户后续再慢慢替换示例内容。

优点：

- 页面看起来更满
- 不需要调整空态逻辑

缺点：

- 会让“项目经历、思考文章、投资记录”看起来像用户已有成果
- 对真实个人站而言，信息失真风险很高

放弃原因：

不符合“少猜测、少假装完成”的仓库原则。

### 方案 B：删掉缺失模块，只保留已有信息

核心思路：
把没有内容的模块完全隐藏，先做成一个更窄的个人页。

优点：

- 信息绝对真实
- 页面不会出现虚构内容

缺点：

- 会破坏上一轮已经建立好的四大模块结构
- 后续重新加回模块时，需要再次改版

放弃原因：

当前站点本来就是为未来扩展而建，直接隐藏模块会损失结构稳定性。

### 方案 C：保留结构，但把缺失模块改为高质量空态

核心思路：
让站点继续保留四大模块，但项目、投资、思考在暂无真实内容时使用明确的“建设中 / 暂未公开 / 即将补充”状态。

优点：

- 保留整体结构完整性
- 不虚构内容
- 为后续补充留出自然入口

缺点：

- 页面内容密度会暂时下降
- 需要专门设计空态文案与展示方式

最终选择原因：

这是“真实性”和“结构延续性”之间最合适的平衡方案。

## 5. 最终决策

本轮最终采用以下策略：

1. 用用户真实资料替换首页和履历主文案。
2. 新增联系方式展示，让站点立刻具备真实人物入口。
3. 删除所有示例文章内容。
4. 项目、投资、思考模块改为设计化空态展示，而不是虚构案例。
5. 保持评论、文章路由、双语框架不删除，因为这些基础设施之后仍然会继续使用。

- 页面表达更诚实，不再依赖示例内容撑场面
- 四大模块结构仍然保留，后续扩展不需要重做架构

改善最明显的质量属性包括：

## 8. 本轮缺点与代价

这一轮的代价和局限也很明确：

1. 页面内容暂时会比上一轮“看起来更空”。
2. 项目、投资、思考三个模块目前主要承担“预留位置”而不是“输出密度”。
3. 由于用户目前提供的信息仍然很少，履历层只能先呈现为方向性介绍，而不是完整 CV。

这些代价是主动接受的，因为它们比“继续用虚构内容冒充完成度”更合理。

## 10. 本轮结论

本轮最重要的结论是：

对于一个刚开始补全真实信息的个人站来说，“保留高级感，但不伪造完成度”是正确策略。

被证实的判断：

- 空态模块并不会削弱站点，反而会提高可信度
- 联系方式和研究方向一旦真实落地，站点的个人识别度会立刻上升
- 先把身份信息写准，比先堆大量内容更重要
