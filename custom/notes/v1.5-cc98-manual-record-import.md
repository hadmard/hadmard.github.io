# 版本记录：v1.5-cc98-manual-record-import

## 1. 本轮目标

本轮目标是基于用户直接提供的 CC98 页面原文，把目标主题真正整理成本地投资记录文件，而不是继续停留在“脚本已搭好但缺授权”的状态。

这一轮属于：功能补充 + 交付补完。

优先级之所以高，是因为上一轮已经把抓取工具基线立住，但用户这次直接给出了完整原文，这意味着当前最合理的动作不再是继续研究抓取，而是立刻把记录产物落地。

## 2. 背景与上下文

本轮承接：

- `custom/notes/v1.4-cc98-investment-crawler.md`

上一轮的关键结论是：

1. 目标主题真实数据走 CC98 鉴权 API。
2. 没有用户 token，不能直接通过 API 完成抓取。
3. 但本地已经有用于沉淀 JSON / Markdown / CSV 的输出结构。

本轮用户进一步提供了该主题的页面原文，因此当前输入条件发生变化：

- 原先缺的是“可访问内容”
- 现在用户已经直接把内容补给了我

这使得任务从“等待授权”切换成了“基于人工来源做一次可信导入”。

## 3. 问题分析

本轮的表面问题是“如何把用户粘贴的原文保存下来”。  
真正的核心矛盾则是：

- 需要尽快形成可用的投资记录产物
- 但当前输入不是结构化 API JSON，而是混杂了页面导航、楼层内容和页脚的原始文本

因此要解决两个问题：

1. 怎么在不引入额外复杂实现的前提下，把文本整理成清晰的记录文件。
2. 怎么明确说明这批数据的来源是“用户提供原文”，而不是“实时授权抓取结果”，避免后续误判其来源和可信边界。

## 4. 候选方案比较

### 方案 A：继续等待 token，什么都不落地

核心思路：

- 保持上一轮脚本不动
- 等用户未来再给 token 后再生成正式记录

优点：

- 数据来源最统一
- 不需要做人工整理

缺点：

- 当前明明已经拿到了原文，却不能产生产物
- 不符合“持续迭代”的仓库要求

放弃原因：

这会让当前这轮输入白白浪费，交付不完整。

### 方案 B：扩展脚本支持“从原文文本自动解析导入”

核心思路：

- 再写一层解析器
- 把用户贴的整段页面文本自动切成楼层

优点：

- 更自动化
- 后续如果还贴同类原文，可以复用

缺点：

- 当前样本只有一份
- 文本混杂导航、楼层、页脚，规则并不稳定
- 为一次导入额外增加复杂解析逻辑，成本偏高

放弃原因：

与当前任务规模不匹配，容易引入不必要复杂度。

### 方案 C：基于用户原文做一次保守的手工结构化导入

核心思路：

- 把原文作为 `manual-source`
- 手工整理出主题元数据和楼层列表
- 直接生成 JSON / Markdown / CSV

优点：

- 最快能形成可用交付物
- 可以明确标注数据来源边界
- 改动集中在输出文件，不扰动已有脚本

缺点：

- 不是自动抓取
- 少量字段需要保守估计，例如未暴露的 postId / userId

最终选择原因：

这是当前最符合“最小改动、立即交付、边界清晰”的方案。

## 5. 最终决策

本轮最终采用方案 C，具体做法如下：

1. 在既有输出结构下新增 `topic-6450962` 的本地记录目录。
2. 保存一份 `manual-source.txt`，明确说明数据来自用户粘贴原文。
3. 补齐：
   - `raw/topic.json`
   - `raw/posts.json`
   - `records/investment-summary.json`
   - `records/investment-record.md`
   - `records/investment-record.csv`
4. 不改上一轮抓取脚本本体，因为当前需求已经能在不扩展代码的前提下完成。

## 7. 本轮优点

当前方案的主要优点：

1. 终于把主题内容真正落成了本地投资记录，而不是只停留在工具层。
2. 输出同时覆盖原始文本、结构化 JSON、Markdown、CSV，便于后续继续加工。
3. 不扩展抓取脚本逻辑，保持了上一轮代码边界稳定。
4. 明确区分了“授权抓取结果”和“用户人工提供原文”，可追溯性更好。

质量属性上的改善主要体现在：

- 交付完整性
- 可复核性
- 可阅读性
- 后续二次加工便利性

## 8. 本轮缺点与代价

本轮也有明确代价：

1. 数据来源不是实时 API，而是用户提供原文，因此不能保证字段百分之百与站点内部结构一致。
2. 部分不可见字段如 `postId`、`userId` 仍然缺失。
3. 楼层切分与摘要中存在少量人工整理成分，后续若拿到 API 数据，仍值得再校对一次。

也就是说，本轮接受的是“先把内容沉淀下来，再等待未来更强的数据源补校验”的取舍。

## 10. 本轮结论

本轮最重要的结论是：

当抓取权限受限、但用户能够直接提供可信原文时，最优先的动作应该是先完成内容沉淀，而不是继续等待更理想的数据入口。

被证实的判断：

- 上一轮建立的输出结构可以直接承接人工导入结果
- 对于当前任务，人工结构化导入已经足够形成可用投资记录
