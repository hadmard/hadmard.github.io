# 版本记录：v1.10-minecraft-learning-map-and-cc98-fix

## 1. 本轮目标
本轮目标是完成首页的“Minecraft 风格”互动学习地图（Learning Map）的深度视觉与交互定制，并修复由于 CC98 爬虫输出的 JSON 结构解析异常导致首页“投资经历”模块消失的 Bug。

## 2. 背景与上下文
用户要求在保留原有节点结构的基础上，把学习路径图升级为丝滑的、支持拖拽的 Minecraft 风格技能树（金矿、钻石、石头等材质），并要求分为“金融的我”和“科研的我”两个核心分支。在完成该地图后，用户反馈首页的投资经历无法显示，经过排查是 src/components/HomePage.astro 在读取 aw/posts.json 时，对 Array.isArray 的判定存在局限，导致 Astro 的构建过程中未成功提取 posts 列表。

## 3. 问题分析
- **UI 更新**：需要引入 @xyflow/react 的高级特性和定制化 Node 渲染，使用 ramer-motion 实现基于状态的面板弹窗展示，同时修改数据源 src/data/learningMap.ts 以贴合双分支设定。
- **CC98 数据展示 Bug**：爬虫生成的 posts.json 数据结构是包含外层数组的 [{ items: [...] }]，但在通过 eadJsonFile<Array<...>> 读取时，原有 Astro 代码没有做健壮的数组判断逻辑。如果解析出来的结构有差异或者类型安全度不够，就会导致 length 检测不通过。

## 4. 候选方案比较
- **UI 更新方案**：
  - 使用标准的 React 组件体系配合硬编码 CSS。
  - 使用 React Flow 结合 Tailwind + Framer Motion，渲染定制化 Minecraft 材质节点。**选择此方案**，因为兼顾了丝滑动画与任意拖拽需求。
- **CC98 Bug 修复方案**：
  - 更新爬虫脚本当中的 JSON 展平逻辑。
  - 更新 HomePage.astro 的 JSON 读取逻辑，增加鲁棒的数组和空值判断。**选择此方案**，因为对源数据的侵入性最小，直接在消费端兼容。

## 5. 最终决策
- 在 src/components/LearningMap.tsx 中全面重构并注入 Minecraft UI 与交互动画。
- 在 src/data/learningMap.ts 补充金融与计算机科研并行的成就分支。
- 在 src/components/HomePage.astro 中利用 Array.isArray(postPagesRaw) 进行强制转换和兜底，确保解析结果总是可用的数组类型。

## 6. 具体实现
- 重写 LearningMap.tsx，定义了 MinecraftNode 自定义组件结构。
- 修改 LearningMap.ts 添加相应的内容节点与连线数据。
- 编写修复脚本，将 HomePage.astro 内 loadSyncedInvestmentFeed 函数下的 postPages 解析逻辑替换为：
  const postPagesRaw = await readJsonFile<any>(path.join(baseDir, 'raw/posts.json'));
  const postPages = Array.isArray(postPagesRaw) ? postPagesRaw : (postPagesRaw ? [postPagesRaw] : []);
- 执行 
pm run build 并确认 HTML 输出包含了原本丢失的“投资经历”模块。

## 7. 本轮优点
- 前端的视觉体验大幅提升，贴合用户“进度条与技能树”的直觉要求。
- 提升了解析脚本的鲁棒性，避免构建过程因为源数据结构的微调而崩溃。

## 8. 本轮缺点与代价
- React Flow 的引入增加了构建产物的包大小。
- 频繁的 AST 和空格陷阱导致修复 Astro 文件时不便直接使用 eplace_string_in_file，依赖 Python 脚本直接覆写文件。

## 9. 验证与结果
- 运行 
pm run build 成功，无报错。
- docs/index.html 的产物中重新捕获到 投资记录已同步到站点首页 文本，验证 Bug 彻底解决。

## 10. 本轮结论
Astro 静态站点构建流程中，对于外部依赖的文件（如 JSON）应采取更加防御性的解析方式，才能避免由于爬虫结构细微变化引发的局部组件非预期隐藏。

- 如有需要，清理或归档旧的爬虫记录以及测试产生的临时文件。
