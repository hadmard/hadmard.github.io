# v1.10 Apple UI Redesign and 404 Pages Fix

## 1. 本轮目标
修复全站 404 的空页面，并将网站整体风格从 Minecraft 像素风/成就系统风格，转型为高级的 Apple/Meta Glassmorphism（毛玻璃）风格。

## 2. 背景与上下文
用户报错在使用过程中遇到了路由导航出现的 404 问题，且对原本的 Minecraft 风格 UI 表示不满意，要求全面“动画拉满、最精密、最丝滑”并参考顶级科技公司如 Meta 和 WorldQuant 的官网。此前投资模块注入记录的数据由于 CDN 机制引起用户误解，验证后确认存在，于是将重心转移至视觉风格革新。

## 3. 问题分析
- `/projects`, `/investments`, `/resume` 路由页面完全缺失，直接导致 404 错误。需要用 Astro 模版系统快速建立占位页面。
- `global.css` 与 `HomePage.astro`, `LearningMap.tsx` 之中硬编码了大量 `.mc-bg`, `.mc-tooltip` 等类，这阻碍了向新视觉系统的转型。需要进行全面清理。
- 需要使用 Tailwind 提供的 `backdrop-blur-*`, `bg-white/[0.05]`, `border-white/10` 等工具类替换色块填充。

## 4. 候选方案比较
- **手动更改**: 逐个文件替换类名和文件结构。缺点是费时且对繁杂样式的剔除容易产生纰漏。
- **自动脚本 (Node.js)**: 使用正则表达式和模板字符串，创建生成脚本并操作 CSS/TSX。可极快地生成六个页面并安全剥离 CSS。

## 5. 最终决策
使用临时构建的 Node.js 脚本 (`scaffold.cjs`, `fix_map.cjs`) 快速完成文件的创立和大量旧类的更新。这可以在不破坏原有组件树逻辑的前提下应用全新玻璃拟态样式。

## 6. 具体实现
- 使用 `scaffold.cjs` 建立 `/pages/projects.astro`, `/pages/investments.astro`, `/pages/resume.astro` 及它们的英文 `/en/` 版本。设计基于玻璃拟态的美观居中结构。
- 取消 `global.css` 中有关 `.mc-` 前缀的所有样式。
- 对 `LearningMap.tsx` 内的背景色、高亮框、阴影，全部替换为通过 `ReactFlow` 加载的深色背景配 `tailwind` 提供的高级拟物玻璃质感 (backdrop-blur-xl)。
- `HomePage.astro` 周围的边框也升级至 3D 悬浮效果。

## 7. 本轮优点
- 极快消灭 404 错误。
- 完成风格切换而不重构底层数据逻辑集，改动小且影响深远。

## 8. 本轮缺点与代价
- 使用正则全局替换时，如果未来引入更新会导致临时脚本不可再用。
- LearningMap 过渡效果（className 等）略有重复冗余（`className="!bg-transparent"`），虽引起构建警告但不影响使用。

## 9. 验证与结果
- 执行 `npm run build` 验证全部页面产出。控制台提示成功生成如 `/en/projects/index.html` 等 8 个页面，之前因为 404 造成的错误得到根本解决。
- 本地验证 UI 已呈现磨砂玻璃的精致效果。

## 10. 本轮结论
新的 Apple/Glassmorphism 视觉基础已经成功扎根在各个主要页面中，为后续的深入个性化展示提供了高尚而成熟的视觉舞台。

- 填充 `/projects` 和 `/resume` 内的真实内容。
