# 版本记录：v1.23-astro-single-site-merge

## 1. 本轮目标

按用户明确要求停止双站并行认知，统一到可部署的 Astro 主站：

1. 仅保留 Astro 作为线上发布口径。
2. 首页补齐 Progress / Projects / Thinking 栏目。
3. 名称统一为 Yfcccc。
4. 保持“输入 GitHub Pages 网址即可看到效果”的路径一致。

## 2. 背景与上下文

本轮先读取：

- `AGENTS.md`
- `custom/notes/v1.22-progress-projects-thinking-and-particles.md`
- `src/components/HomePage.astro`
- `src/components/SiteHeader.astro`
- `src/data/site.ts`
- `README.md`

延续思路：

1. 保持 Astro 构建到 `docs/` 的发布流程。
2. 保留进度地图作为核心展示模块。

放弃思路：

1. 放弃由 Next 子站承载改动的方案。
2. 放弃“主站不改、只改实验目录”的做法。

## 3. 问题分析

用户“看不到变化”的根因是部署口径错位：

1. 线上页面来自 Astro 主站，而近期改动集中在 Next 子目录。
2. Astro 首页缺少独立 Projects/Thinking 栏目与目标项。
3. 导航原先偏页面跳转，不利于回到 progress 锚点。

## 4. 候选方案比较

方案 A：继续维护双站，文档解释差异。  
优点：改动少。  
缺点：用户持续混淆，线上感知弱。

方案 B：把关键需求全部落在 Astro 主站。  
优点：部署路径唯一、效果可直接在 GitHub Pages 看到。  
缺点：需要重复迁移部分能力。

最终采用方案 B。

## 5. 最终决策

1. Astro 首页直接补齐 Progress / Projects / Thinking。
2. 导航改为首页锚点优先（progress/projects/thinking）。
3. 品牌名与 SEO 文案统一 Yfcccc。
4. README 明确“仅 Astro 参与发布”。
