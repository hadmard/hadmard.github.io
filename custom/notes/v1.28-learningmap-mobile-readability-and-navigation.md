# 版本记录：v1.28-learningmap-mobile-readability-and-navigation

## 1. 本轮目标

在 v1.27 基础上继续精修 `LearningMap` 的移动端体验，确保手机端可读性与可操作性达到可发布标准。

## 2. 背景与上下文

本轮先读取：

- `AGENTS.md`
- `custom/notes/v1.27-learningmap-internal-apple-alignment.md`
- `src/components/LearningMap.tsx`

延续思路：

1. 保持 Apple 风格颜色语义与组件结构。
2. 保持进度树核心交互：悬停说明、点击详情、依赖可视化。

放弃思路：

1. 放弃桌面优先参数直接套用到移动端。
2. 放弃移动端完全不可拖拽浏览的交互限制。

## 3. 问题分析

v1.27 后在手机端仍有可用性问题：

1. 节点宽度和文字密度偏大，阅读压迫。
2. 缩放下限偏高，视口中难以一次看到关键关系。
3. 触屏下无法拖拽画布，浏览长图谱成本高。
4. 弹窗与角标在小屏上占比偏大。

## 4. 候选方案比较

方案 A：仅调小字号。  
优点：改动小。  
缺点：无法解决导航与缩放问题。

方案 B：同时调整节点尺寸、缩放策略、触摸交互与弹窗排版。  
优点：完整解决可读性与可操作性。  
缺点：需要增加一层响应式状态判断。

最终采用方案 B。

## 5. 最终决策

1. 增加 `isCompact` 响应式状态，按屏宽切换移动端参数。
2. 移动端开启画布拖拽与双指缩放。
3. 节点、悬浮说明、图例、弹窗统一做小屏缩放适配。

## 6. 具体实现

文件：`src/components/LearningMap.tsx`

核心改动：

1. 引入 `useEffect` 监听窗口宽度，维护 `isCompact`。
2. 节点卡片宽度改为 `184px`（小屏）到 `214px`（大屏）响应式。
3. 标题字号和悬浮说明宽度增加小屏分档。
4. 画布高度小屏降至 `600px`，信息角标与图例缩小。
5. ReactFlow 参数按屏幕切换：
   - `panOnDrag`：小屏开启
   - `zoomOnPinch`：小屏开启
   - `minZoom/maxZoom/fitViewOptions`：小屏放宽缩放范围
6. 详情弹窗标题、间距、行高改为小屏友好值。

## 7. 本轮优点

1. 手机端首次进入能看到更完整的图谱关系。
2. 触控交互更自然，可拖拽浏览长图。
3. 文本与面板密度下降，阅读成本更低。

## 8. 本轮缺点与代价

1. 增加了一个基于窗口宽度的状态分支，组件复杂度略升。
2. 小屏可缩放范围放宽后，需要后续真实设备上再观察误触概率。
