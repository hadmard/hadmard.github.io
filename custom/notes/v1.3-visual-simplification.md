# 版本记录：v1.3-visual-simplification

## 1. 本轮目标

本轮目标是根据用户最新反馈，把首页从“视觉元素偏多、信息偏挤、装饰感偏强”的状态，调整为更素、更安静、更像研究者个人主页的风格。

这一轮属于：可维护性提升 + 表达风格修正。

优先级之所以高，是因为当前问题不在功能缺失，而在第一眼感受已经偏离用户预期。如果视觉气质不对，后续继续填内容只会放大问题。

## 2. 背景与上下文

本轮承接：

- `custom/notes/v1.2-pages-branch-deploy.md`

在上一轮中，站点已经切换为 `main /docs` 的 GitHub Pages 分支部署方式，部署链路已经稳定。

但视觉层仍保留了此前较强的“品牌感”和“氛围感”设计思路，包括：

- 背景光斑和网格装饰
- 较多半透明卡片与高亮边框
- 首页信息块数量偏多
- Hero 区和导航区的视觉密度较高

这些做法在“个性化展示页”语境下并非错误，但用户明确反馈“太难看了，太拥挤了”“花里胡哨，素一点”，说明当前重点不再是强化炫酷感，而是收敛风格、降低噪声。

## 3. 问题分析

表面问题是“不好看、太拥挤”，但核心矛盾其实是：

1. 视觉层次太多，导致主信息不够突出。
2. 装饰性元素抢占了内容本身的注意力。
3. 当前页面更像“展示型 landing page”，而不是“克制型个人主页”。

进一步拆解后，主要问题集中在三个方面：

- 背景层过于活跃：放射渐变、网格、玻璃态叠加，造成页面一直在“说话”。
- 模块拆分过细：同类信息被拆成多个卡片，阅读时需要频繁切换注意力。
- 对比度和强调点过多：标题、按钮、标签、卡片边框都在争夺视觉主导权。

因此，这一轮不需要重做结构逻辑，而需要做一次明确的“降噪”。

## 4. 候选方案比较

### 方案 A：只做局部减法

核心思路：
仅删掉少量装饰，例如背景特效、部分标签、个别卡片阴影，其余结构保持不变。

优点：

- 改动最小
- 风险最低
- 不容易影响现有布局逻辑

缺点：

- 页面骨架依然偏拥挤
- 信息密度问题不会真正解决
- 容易变成“去掉一点装饰，但本质没变”

放弃原因：

用户反馈已经不是“稍微收一收”，而是明确表示整体审美方向不对。只做局部减法，大概率不够。

### 方案 B：保留内容结构，重做首页表达方式

核心思路：
不改站点模块边界与双语结构，只对首页的版式、背景、卡片密度、导航表现和色彩强调方式做整体收敛。

优点：

- 能保留现有内容组织和部署链路
- 能显著改善拥挤感
- 改动集中在视觉层，后续继续填内容时更稳

缺点：

- 改动范围比局部微调更大
- 需要重新生成 `docs/`
- 需要重新验证首页结构是否仍然完整

最终选择原因：

该方案在“保留系统能力”和“纠正页面气质”之间取得了更好的平衡，能够直接回应用户反馈。

## 5. 最终决策

本轮最终采用方案 B，具体策略如下：

- 去掉主动吸引注意力的背景装饰层
- 把首页从“多块展示卡片”收拢为“简洁研究者主页”布局
- 减少首页首屏展示的信息数量
- 用留白、细边框和克制配色代替大面积高亮效果
- 保留双语切换、四大模块结构和分支部署方式，不动站点基础能力

换句话说，这一轮不是推翻前面的工程，而是把视觉表达从“炫酷展示”校正到“安静可信”。

## 6. 具体实现

本轮主要改动文件如下：

### `src/layouts/Layout.astro`

作用：

- 删除全局背景中的放射渐变层与网格层
- 让页面回归更干净的基础布局

### `src/styles/global.css`

作用：

- 重写全局配色变量，改为浅底、低对比、轻阴影的体系
- 改用更克制的组件样式，弱化玻璃态和重装饰感
- 简化常用类，如 `panel`、`panel-strong`、`soft-label`、`nav-link`、`metric-card` 等
- 让页面视觉语言更统一，不再依赖大面积氛围特效

### `src/components/HomePage.astro`

作用：

- 重新组织首页结构，减少首屏卡片数量
- 取消过于显眼的悬浮导航表现
- Hero 区改成更简洁的双栏：主体介绍 + 右侧简洁信息卡
- 各模块改为更平稳的卡片和留白关系
- 保留中英双语、简历/项目/投资/思考四大模块与空态逻辑

## 7. 本轮优点

相比上一版，本轮的主要改进是：

1. 首屏更安静，用户身份信息终于成为视觉中心。
2. 阅读路径更顺，不再需要在多个强调点之间来回跳。
3. 页面整体更贴近“研究者主页”而不是“产品营销页”。
4. 后续继续填真实内容时，页面承载力更强，不容易因为内容增加而继续显得拥挤。

质量属性上的改善主要体现在：

- 可读性
- 可维护性
- 风格一致性
- 内容承载稳定性

## 8. 本轮缺点与代价

这一轮也有明确代价，不能忽略：

1. 个性化视觉冲击力明显下降，页面第一眼不会像上一版那样“抓人”。
2. 当前版本更依赖真实内容质量，如果内容长期空置，会显得偏素甚至偏空。
3. 为了快速纠偏，本轮主要处理的是首页，尚未继续细化更深层的内容表达方式。

也就是说，这一轮是主动接受“少一点惊艳，换更高耐看度”的取舍。

## 9. 验证与结果

本轮已完成以下验证：

- `npm run check`
- `npm run build`

验证结果：

- 构建通过
- `docs/` 目录正常生成
- 中英文首页仍可正常输出

已知提示：

- `src/content/thoughts` 目录暂无文章文件，因此 Astro 会提示未匹配到 `md/mdx` 内容，这是当前空态设计下的预期行为
- 构建中仍有一条 Astro/Vite 上游未使用导入 warning，不影响静态产物输出

## 10. 本轮结论

本轮最重要的结论是：

对这个站点来说，当前阶段更重要的不是“风格记忆点最大化”，而是“页面气质与本人身份一致”。  
用户已经明确偏好更素、更干净、更可信的表达，因此后续设计应默认继承“克制优先、装饰次之”的原则。

被证实的判断：

- 视觉减法能明显改善拥挤感
- 结构不必大改，主要矛盾在表现层

被修正的旧判断：

- 上一版默认“更强的氛围感会提升个性表达”，这对当前用户并不成立

## 11. 下一轮建议

下一轮最值得继续做的事情：

1. 继续补真实项目内容，这是当前收益最高的动作。
2. 为“项目经历”和“个人思考”设计更克制的内容模板，避免未来内容一多又重新变挤。
3. 若用户愿意，再补一张合适的头像或个人照片，用低干扰方式增强识别度。
