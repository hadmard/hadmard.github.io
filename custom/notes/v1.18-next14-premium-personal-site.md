# 版本记录：v1.18-next14-premium-personal-site

## 1. 本轮目标

在现有仓库内新增一个独立 Next.js 14（App Router）+ TypeScript + TailwindCSS 的个人网站项目，满足用户给出的高端极简设计规范与首页模块要求。

## 2. 背景与上下文

本轮先读取：

- `AGENTS.md`
- `custom/notes/v1.17-learning-map-sections-and-project-row.md`
- `custom/notes/v1.16-learning-map-mc-style-static.md`

本轮定位：

1. 这是新技术栈需求，与当前 Astro 主站并行，不应直接覆盖原站。
2. 需要完整项目结构、可复用组件、可生产构建代码。

## 3. 问题分析

用户要求同时覆盖设计哲学、技术栈、版块结构、动画边界与工程可维护性，问题核心是：

1. 不能做模板化作品，必须有系统化视觉与结构控制。
2. 动效必须“少而准”，不能喧宾夺主。
3. 页面模块必须可复用，并且数据与视图解耦。

## 4. 候选方案比较

方案 A：在 Astro 站点上局部改造模拟 Next 风格。  
优点：改动少。  
缺点：不满足用户明确技术栈要求（Next.js 14 App Router）。

方案 B：在仓库内新建独立 Next 子项目。  
优点：完整满足技术栈与工程边界，且不影响现有站点。  
缺点：文件新增较多。

最终采用方案 B。

## 5. 最终决策

1. 在 `next-premium-site/` 下创建独立项目。
2. 首页拆分为 5 个可复用 section 组件。
3. 使用黑白灰 + 单一蓝色强调，控制动效在允许范围。
4. 版本历史用时间线结构并覆盖 version/goal/changes/reasoning/next step。

## 6. 具体实现

### 6.1 工程结构

新增目录与文件：

- `next-premium-site/app/`
- `next-premium-site/components/sections/`
- `next-premium-site/components/ui/`
- `next-premium-site/data/`
- `next-premium-site/package.json`
- `next-premium-site/tailwind.config.ts`
- `next-premium-site/tsconfig.json`
- `next-premium-site/next.config.mjs`
- `next-premium-site/postcss.config.js`

### 6.2 可复用组件拆分

- `HeroSection`
- `ProjectShowcaseSection`
- `MethodologySection`
- `VersionHistorySection`
- `AboutSection`
- `SectionReveal`（滚动渐入）

### 6.3 设计规则落地

1. 仅使用黑/白/灰 + 蓝色强调色（`accent: #2D6BFF`）。
2. 大字号层级与宽松留白（section 基本 `py-20` 起）。
3. 避免边框依赖，主要通过间距与层级表达。
4. 字体控制在 2 套：Sora（展示）+ IBM Plex Sans（正文）。

### 6.4 动效规则落地

1. `SectionReveal` 使用 `opacity + translateY` 的滚动渐入。
2. 项目卡 hover 仅做 `scale 1.02` + 阴影增强 + 额外信息展开。
3. Hero 背景使用轻量渐变位移动画，无跳跃、无滥用视差。

### 6.5 内容结构

- Hero：大标题、简述、CTA、极简动态背景。
- Project Showcase：卡片网格，含标题/描述/tags/额外信息。
- Methodology：系统化流程说明。
- Version History：时间线条目完整字段。
- About：简洁可读。

## 7. 本轮优点

1. 完整满足用户技术栈与结构要求。
2. 组件边界清晰，便于后续扩展和维护。
3. 视觉与动效克制，符合高端极简方向。

## 8. 本轮缺点与代价

1. 新增为独立子项目，和主站内容同步需要后续流程约定。
2. 当前文案为高质量基线，后续仍建议按个人经历继续定制。
