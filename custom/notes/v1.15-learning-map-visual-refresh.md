# 版本记录：v1.15-learning-map-visual-refresh

## 1. 本轮目标

针对用户反馈“进度地图没设计感、没新意”，在保持现有交互逻辑不变的前提下，重做学习地图的视觉表达与信息层次。

## 2. 背景与上下文

本轮先读取并延续：

- `custom/notes/v1.14-google-apple-contrast-fix.md`
- `src/components/LearningMap.tsx`
- `src/components/HomePage.astro`

当前问题集中在地图视觉：

1. 画布过于平，层次与氛围不足。
2. 节点状态区分不够鲜明，视觉记忆点弱。
3. 缺少“快速读懂状态”的图例辅助。

## 3. 问题分析

原版虽可读，但设计语言偏保守：

1. 节点主要是灰蓝浅色，状态差异小。
2. 背景只有基础点阵，不足以形成视觉中心。
3. 首屏说明与地图融在一起，缺乏“信息卡片”质感。

## 4. 候选方案比较

方案 A：仅替换背景图。

- 优点：改动少。
- 缺点：版权与风格一致性风险高，且不能从结构上提升地图设计。

方案 B：保持代码可维护，做原创视觉重构（画布层、节点主题、图例、弹窗头部语义色）。

- 优点：版权风险低，可控且一致。
- 缺点：需要改动组件多个区域。

本轮采用方案 B。

## 5. 最终决策

1. 使用原创渐变光晕层替代外部下载图片。
2. 节点状态采用更强语义色：
   - 完成：emerald
   - 进行中：blue
   - 可开始：amber
   - 锁定：slate
3. 增加底部图例，降低首次理解成本。
4. 保留 ReactFlow 交互、边关系、点击详情逻辑。

## 6. 具体实现

文件：`src/components/LearningMap.tsx`

### 6.1 画布层次升级

- 引入 `BackgroundVariant` 并调整点阵密度。
- 增加三层模糊色块（蓝/绿/橙）营造空间感。
- 标题说明改为半透明白卡片，增强“Apple 式简洁表层”质感。

### 6.2 节点视觉重构

- 完成节点：`emerald` 卡片与阴影。
- 进行中节点：`blue` 卡片，并增加脉冲氛围。
- 可开始节点：`amber` 卡片。
- 锁定节点：低对比灰。

### 6.3 连线语义增强

- 连线颜色按目标节点状态映射语义色。
- 未完成依赖仍保留浅灰弱化。

### 6.4 信息辅助

- 左上说明改卡片化。
- 新增左下状态图例（4 种状态），提升可扫读性。

### 6.5 详情弹窗样式统一

- 弹窗头部根据状态变更底色与边框。
- 状态标签颜色与节点语义色一致。

## 7. 本轮优点

1. 视觉层次明显提升，地图更有设计感。
2. 状态认知更快，阅读路径更清晰。
3. 无外部图片依赖，版权与维护风险更低。

## 8. 本轮缺点与代价

1. 仍基于手工节点坐标，布局策略本身未重构。
2. 视觉增强后，如后续继续加节点，需控制色彩密度避免过载。

## 9. 验证与结果

已执行：

- `npm run build`

预期：

## 10. 本轮结论

本轮通过原创视觉重构，解决了“没新意、没设计感”的核心反馈，并保持了现有功能与信息结构稳定。
