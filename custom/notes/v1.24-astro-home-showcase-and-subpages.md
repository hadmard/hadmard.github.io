# 版本记录：v1.24-astro-home-showcase-and-subpages

## 1. 本轮目标

按用户要求重构单站层次：

1. 首页变为纯展示页（不放实质内容）。
2. 右上角栏目全部跳转到独立子页面。
3. 进度单独页面承载横向（左到右）结构。
4. 实习目标放入进度页面。
5. 保留炫酷首页动效。

## 2. 背景与上下文

本轮先读取：

- `AGENTS.md`
- `custom/notes/v1.23-astro-single-site-merge.md`
- `src/components/HomePage.astro`
- `src/components/SiteHeader.astro`
- `src/data/site.ts`
- `src/pages/*`

延续思路：

1. 继续使用 Astro 单站发布到 `docs/`。
2. 保留首页粒子动画作为展示层体验。

放弃思路：

1. 放弃在首页堆叠 Progress / Projects / Thinking 实质内容。
2. 放弃首页锚点导航为主的结构。

## 3. 问题分析

用户反馈“层次非常乱”的核心原因：

1. 首页承担了过多内容，展示层与内容层混杂。
2. 导航以锚点为主，不能形成清晰子页面信息架构。
3. 进度与实习目标的关系没有在独立页面中完整表达。

## 4. 候选方案比较

方案 A：继续首页聚合内容，仅做样式整理。  
优点：改动小。  
缺点：层次问题无法根治。

方案 B：首页展示层 + 子页内容层分离。  
优点：结构清晰，访问路径明确，便于持续扩展。  
缺点：需要新增页面与导航改造。

最终采用方案 B。

## 5. 最终决策

1. 首页仅保留网站定位说明 + 粒子动效 + 入口按钮。
2. 导航统一跳转子页面（非锚点）。
3. 新增 `progress` 子页面（中英文）承载地图与实习目标汇总。
4. 新增 `thinking` 子页面（中英文）承载 test 占位条目。

## 7. 本轮优点

1. 首页层次显著清晰：只展示，不承载细节内容。
